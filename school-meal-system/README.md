## Node
**Для начала разработки, нам нужно установить Node** - это открытая кроссплатформенная среда выполнения, позволяющая запускать код JavaScript на сервере, а не только в браузере. Построенная на движке Chrome V8, она превращает JavaScript в язык общего назначения, поддерживая асинхронную модель ввода-вывода, что обеспечивает высокую производительность при разработке бэкенда

## Npm
**NPM (Node Package Manager)** - это стандартный менеджер пакетов для среды выполнения JavaScript Node.js, автоматически устанавливаемый вместе с ним. Он позволяет разработчикам легко устанавливать, обновлять, делиться и управлять библиотеками (пакетами/модулями) в проектах, используя командную строку (CLI)

Через IDE (желательно VS code) зайдите в папку с репозитория. Там скорее всего пока что будут лежать README.md и папка веб-приложения.

Через терминал зайдите в папку веб-приложение и скачайте все зависимости:
```bash
# Находясь в корневой папке репозитория
cd school-meal-system
npm install
```
Оно сразу поставит вам всё что нужно, осталось только разобраться с полученными инструментами.

Первое, что вы захотите сделать, используя npm - запустить сайт:
```bash
npm run dev # В папке веб-приложения
```
Пока вы не прервали действие этой команды, ваш сайт будет работать по локальному пути и динамически обновляться. То есть, пока эта команда работает, вы можете менять код веб-приложения, и вам даже не придется обновлять страницу, чтобы посмотреть изменения

## React
**React** - это популярная JavaScript-библиотека с открытым исходным кодом, разработанная Meta (Facebook) для создания динамических пользовательских интерфейсов (UI). Она позволяет разбивать интерфейс на компоненты, обеспечивая высокую скорость рендеринга за счет использования Virtual DOM. Основная роль React — эффективное управление отображением данных, что делает его идеальным для одностраничных (SPA) и мобильных приложений.

**Vite React** - это современный, чрезвычайно быстрый инструмент сборки и сервер разработки (dev server) для создания приложений на React. Он заменяет классические сборщики вроде Webpack или Create React App (CRA), обеспечивая мгновенную горячую перезагрузку модулей (HMR) за счет использования встроенных в браузер ES-модулей.

Мы если что используем Vite React, пользуясь преимуществами крутого сборщика Vite. Ну это фан информация, особо не забивайте голову.

Мы будем делать всё по чень простой структуре: `App` - точка, где будут отображаться все страницы, хеадеры, футеры. `pages/` - папка для страницы, `components/` - папка для компонентов. Скоро добавлю сюда ещё информацию по структуризации бека для базы данных.

!Мы пишем на JavaScript! Сейчас более модно писать на TypeScript, но проще на обычном js, так что учтите это и не запутайтесь

Остальное сами знаете у кого спросить.

## Tailwind
**Tailwind CSS** - это утилитарный (utility-first) CSS-фреймворк, позволяющий создавать уникальные интерфейсы прямо в HTML-разметке с помощью готовых низкоуровневых классов (например, flex, pt-4, text-center). В отличие от Bootstrap, он не предоставляет готовых компонентов (кнопок, навбаров), а дает гибкие инструменты для быстрой верстки без написания CSS в отдельных файлах.

[Дока](https://tailwindcss.com/)

Проще говоря, так вы сможете быстро писать стили для странички

Пример:
```html
<div class="flex flex-col items-center gap-6 p-7 md:flex-row md:gap-8 rounded-2xl">
  <div>
    <img class="size-48 shadow-xl rounded-md" alt="" src="/img/cover.png" />
  </div>
  <div class="flex items-center md:items-start">
    <span class="text-2xl font-medium">Class Warfare</span>
    <span class="font-medium text-sky-500">The Anti-Patterns</span>
    <span class="flex gap-2 font-medium text-gray-600 dark:text-gray-400">
      <span>No. 4</span>
      <span>·</span>
      <span>2025</span>
    </span>
  </div>
</div>
```

Кроме того, с ним очень удобно настраивать адаптивность. Ну с этим всем остальным я думаю вы сами разберетесь.

## DaisyUI
**DaisyUI** - это популярный плагин-библиотека компонентов для Tailwind CSS, который добавляет семантические имена классов (например, btn, card) к утилитарным классам Tailwind. Она позволяет создавать компоненты интерфейса быстрее, уменьшая количество CSS-кода

[Дока](https://daisyui.com/?lang=ru)

Проще говоря, готовые шаблоны для разных объектов. К тому же оно полностью совместимо с Tailwind. В нём хранится сразу полная тема, так что все компоненты сразу будут подходить под стиль, а так же будут специальные классы, которые задают цвет из темы.

## Prettier
Prettier - это популярный инструмент (opinionated code formatter), предназначенный для автоматического форматирования исходного кода. Он анализирует код и переписывает его с соблюдением единого стиля, автоматически расставляя отступы, кавычки, переносы строк и запятые, что исключает споры о стиле оформления в команде и улучшает читаемость.

Всё что вам стоит знать о нём в нашем проекте:
```bash
npm run format # В папке веб-приложения
```

Полностью приведет все файлы с кодом к одному красивому стилю.ж

## Eslint
**ESLint** - это популярный инструмент статического анализа кода для JavaScript и TypeScript, который находит синтаксические ошибки, баги и нарушения стиля до выполнения программы

Всё что вам стоит знать о нём в нашем проекте:
```bash
npm run lint # В папке веб-приложения
```

Выведет в консоль ошибки и предупреждения, по типу неиспользуемых переменных
